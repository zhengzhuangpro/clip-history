# 架构设计

## 整体架构

Clip History 采用经典的前后端分离架构，通过 Tauri IPC 桥接：

```
┌─────────────────────────────────────────────────┐
│                    用户界面                       │
│           React + TypeScript + Tailwind          │
│                                                  │
│  ┌──────────┐ ┌──────────┐ ┌──────────────┐     │
│  │ 列表视图  │ │ 搜索栏   │ │ 预览面板     │     │
│  └─────┬────┘ └─────┬────┘ └──────┬───────┘     │
│        └─────────────┼─────────────┘             │
│                      │                           │
│              ┌───────┴───────┐                   │
│              │  Zustand Store │                   │
│              └───────┬───────┘                   │
│                      │  invoke / listen           │
├──────────────────────┼───────────────────────────┤
│                Tauri IPC Bridge                   │
├──────────────────────┼───────────────────────────┤
│                      │                           │
│              ┌───────┴───────┐                   │
│              │  Rust 后端     │                   │
│              │               │                   │
│  ┌───────────┼───────────────┼───────────┐       │
│  │           │               │           │       │
│  │  剪贴板   │   数据库层    │  系统集成  │       │
│  │  监听器   │   (SQLite)    │           │       │
│  │           │               │  · 托盘    │       │
│  │  · 文本   │  · 历史记录   │  · 快捷键  │       │
│  │  · 图片   │  · 搜索索引   │  · 窗口    │       │
│  │           │               │           │       │
│  └───────────┴───────────────┴───────────┘       │
│                                                  │
└─────────────────────────────────────────────────┘
```

## 数据流

### 剪贴板监听流程

```
系统剪贴板变化
    │
    ▼
Rust 监听线程（轮询 / 系统事件）
    │
    ├── 检测内容是否与上次相同（去重）
    │
    ▼
提取内容（文本 / 图片 Blob）
    │
    ├── 文本：直接存储 UTF-8 字符串
    ├── 图片：压缩 + 生成缩略图 → 存储 Blob
    │
    ▼
写入 SQLite
    │
    ▼
通过 Tauri Event 推送通知前端
    │
    ▼
前端 Zustand Store 更新 → UI 刷新列表
```

### 用户操作流程

```
用户点击历史条目
    │
    ▼
前端调用 tauri.invoke('copy_to_clipboard', { id })
    │
    ▼
Rust 后端从 SQLite 读取内容
    │
    ▼
写入系统剪贴板
    │
    ▼
返回成功/失败 → 前端显示操作反馈
```

## 关键模块设计

### 剪贴板监听器（clipboard/）

- 使用 `arboard` crate 读取系统剪贴板
- 轮询间隔：200ms（可配置），检测到变化才触发处理
- 去重：对比最近一条记录的内容哈希
- 图片处理：超过一定尺寸自动压缩，生成缩略图用于列表显示

### 数据库层（db/）

- 使用 `rusqlite` + `r2d2` 连接池
- 初始化时自动创建表和索引
- 支持迁移脚本，保证版本升级平滑
- 关键查询使用 Prepared Statement 缓存优化

### 系统集成

- **全局快捷键**：使用 Tauri 2 的 GlobalShortcut API
- **系统托盘**：使用 Tauri 2 的 TrayIcon API，右键菜单包含：显示窗口、设置、退出
- **窗口管理**：快捷键唤起时窗口置顶并聚焦，失焦自动隐藏

### 前端架构

- **列表虚拟化**：历史记录可能很长，使用虚拟滚动保证性能
- **搜索**：前端模糊匹配（小数据量）+ 后端 SQL LIKE（大数据量）双策略
- **主题**：支持亮色/暗色，跟随系统或手动切换

## 性能考量

| 场景 | 策略 |
|------|------|
| 剪贴板监听 | 仅在内容变化时处理，避免无意义写入 |
| 列表渲染 | 虚拟滚动，仅渲染可视区域 |
| 图片显示 | 缩略图缓存，原图按需加载 |
| 搜索 | 防抖 300ms，减少无效查询 |
| 数据库 | 连接池 + Prepared Statement 缓存 |
| 内存占用 | 图片缩略图 LRU 缓存，限制最大缓存数 |
